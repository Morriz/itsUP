# Proxy Stack

The proxy stack provides reverse proxy, TLS termination, and routing capabilities using Traefik v3.6.17.

## Components

### Traefik (v3.6.17)

Pinned in `projects/itsup.yml` (`versions.traefik`). v3.5.x is incompatible with
modern Docker daemons (it hardcodes Docker API v1.24, which daemons >=27.x reject).

**Purpose**: Layer 7 HTTP reverse proxy and Layer 4 TCP passthrough for TLS services.

**Key Features**:
- Automatic TLS certificate management (Let's Encrypt)
- Dynamic service discovery via Docker labels
- TCP passthrough for services requiring end-to-end TLS
- Access logging with rotation
- Middleware support (authentication, rate limiting, headers)

**Configuration**:
- Base config: `proxy/traefik/traefik.yml` (generated by `bin/write_artifacts.py` from `tpl/traefik.yml.j2`, trusted IPs from `lib/data.py:get_trusted_ips()`)
- User overrides: `projects/traefik.yml` (deep-merged on top)
- Dynamic config: `proxy/traefik/dynamic/{middlewares,routers-http,routers-tcp,routers-udp}.yml`

**Entrypoints**:
- `web` (:8080): HTTP entry. The LAN router port-forwards external :80 here.
- `web-secure` (:8443): HTTPS entry with TLS termination. The router port-forwards external :443 here.
- Dynamic `{tcp|udp}-{hostport}` entrypoints: generated per project TCP/UDP ingress (`tpl/traefik.yml.j2`), e.g. for an SNI-routed passthrough broker needing its own port.

**Trusted IPs**:
- Only the router IP as a `/32` (from `routerIP` in `projects/itsup.yml`), produced by `lib/data.py:get_trusted_ips()`.

This is used for `forwardedHeaders` and `proxyProtocol` to trust X-Forwarded-* headers.

### Socket proxy (`proxy_docker`)

**Image**: `wollomatic/socket-proxy:1` (host network, container_name `proxy_docker`).

**Purpose**: Secure Docker socket proxy to limit Traefik's access to the Docker API.

**Why Needed**: Traefik needs Docker API access for service discovery but should not have full socket access (security risk). Traefik reaches it via `DOCKER_HOST=tcp://127.0.0.1:2375`.

**Access Restrictions**:
- Read-only socket mount (`/var/run/docker.sock:ro`), `cap_drop: ALL`, `no-new-privileges`
- GET-only allowlist: `version`, `containers/.*`, `events.*` (`-allowGET` regex)
- `-allowfrom=127.0.0.1` (loopback only)

**Configuration**: `proxy/docker-compose.yml`

### CrowdSec (`proxy_crowdsec`)

**Image**: `crowdsecurity/crowdsec` (version pinned in `projects/itsup.yml` `versions.crowdsec`).

**Purpose**: Collaborative threat detection. Reads Traefik access logs and feeds ban decisions to the Traefik bouncer plugin (`crowdsec-bouncer-traefik-plugin`, declared in `tpl/traefik.yml.j2`).

**Conditional**: Only included when `crowdsec.enabled: true` in `projects/itsup.yml`. When enabled, Traefik `depends_on` it being healthy. Listens on loopback only (`127.0.0.1:7422` AppSec, `127.0.0.1:18080` API).

## Deployment

### Start/Stop
```bash
itsup proxy up           # Start all services
itsup proxy up traefik   # Start only Traefik
itsup proxy down         # Stop all services
itsup proxy restart      # Restart all services
itsup proxy logs         # Tail all logs
itsup proxy logs traefik # Tail Traefik logs only
```

### Dependencies
- **Requires**: DNS stack (proxynet network)
- **Required by**: All upstream projects (need Traefik for routing)

### Secrets
Loaded from `secrets/itsup.txt` (shared infrastructure secrets).

## Routing Configuration

Upstream projects define routing in `projects/{project}/itsup-project.yml`:

```yaml
enabled: true
ingress:
  - service: web
    domain: my-app.example.com
    port: 3000
    router: http  # or tcp
    middleware: []  # Optional Traefik middlewares
```

This generates Traefik labels in `upstream/{project}/docker-compose.yml`:

```yaml
labels:
  - traefik.enable=true
  - traefik.http.routers.{project}-{service}.rule=Host(`{domain}`)
  - traefik.http.routers.{project}-{service}.tls.certresolver=letsencrypt
  - traefik.http.services.{project}-{service}.loadbalancer.server.port={port}
```

### TCP Passthrough

For services that need end-to-end TLS (no termination at Traefik):

```yaml
ingress:
  - service: web
    domain: example.com
    port: 8123
    router: tcp
    tls:
      passthrough: true
```

Generates:
```yaml
labels:
  - traefik.tcp.routers.{project}-{service}.rule=HostSNI(`{domain}`)
  - traefik.tcp.routers.{project}-{service}.tls.passthrough=true
  - traefik.tcp.services.{project}-{service}.loadbalancer.server.port={port}
```

## Host-Only Projects

Some projects run on the host (not containerized) but need Traefik routing:

```yaml
# projects/itsup/itsup-project.yml
enabled: true
host: 127.0.0.1  # Host IP (loopback for the on-host API server)
ingress:
  - domain: itsup.srv.instrukt.ai
    port: 8888
    router: http
```

These projects:
- Have no `docker-compose.yml` in `projects/` directory
- Skip artifact generation in `bin/write_artifacts.py`
- Skip deployment in `lib/deploy.py`
- Still generate Traefik routing config in `proxy/traefik/dynamic/routers-http.yml` (backend `http://{host}:{port}/`)

## Logging

### Access Logs
- **File**: `logs/access.log`
- **Format**: JSON (`accessLog.format: json` in `tpl/traefik.yml.j2`)
- **Rotation**: logrotate (10M, 5 rotations, compressed)
- **Rotation Method**: USR1 signal to Traefik (zero-downtime log reopening)

See [Logging Documentation](../operations/logging.md) for details.

### Container Logs
- **Driver**: json-file
- **Max Size**: 10M per file
- **Max Files**: 3 rotations
- **View**: `itsup proxy logs` or `docker compose logs`

## Troubleshooting

### Certificate Issues

**Problem**: Let's Encrypt rate limit exceeded
```bash
# Check certificate status
itsup proxy logs traefik | grep -i "certificate"

# Remove stuck certificates (forces renewal)
rm data/acme/acme.json
itsup proxy restart
```

### Socket proxy / cert renewals silently failing

If `proxy_docker` and `proxy-traefik-1` show `unhealthy`, Traefik logs
`127.0.0.1:2375: i/o timeout`, and public HTTPS returns `tlsv1 unrecognized name`,
the OpenSnitch verdict path has wedged loopback TCP. Root fix:
`sudo systemctl restart opensnitch` (restarting only the socket proxy or only
Traefik is insufficient). See
[Networking → OpenSnitch loopback wedge](../networking.md#opensnitch-loopback-wedge-traefik-cant-reach-the-socket-proxy).

### Service Not Reachable

**Problem**: Upstream service not accessible via domain

**Check**:
1. Verify itsup-project.yml configuration: `cat projects/{project}/itsup-project.yml`
2. Check generated labels: `grep -A 20 "labels:" upstream/{project}/docker-compose.yml`
3. Verify Traefik sees the service: `itsup proxy logs traefik | grep {project}`
4. Check service is running: `itsup svc {project} ps`

### Trusted IPs Issues

**Problem**: X-Forwarded-* headers not being trusted

**Check**:
```bash
# Verify current trusted IPs
grep -A 1 "trustedIPs:" proxy/traefik/traefik.yml

# Should show only the router IP as /32, e.g.:
# trustedIPs: ['192.168.1.1/32']
```

If the IP is wrong, router IP detection may be incorrect. Check the `routerIP` setting in `projects/itsup.yml`.

## Security

### Docker Socket Proxy
- Traefik never has direct Docker socket access
- All Docker API requests go through the socket proxy `proxy_docker` (read-only, GET-only allowlist: version/containers/events)
- Reduces attack surface if Traefik is compromised

### TLS Configuration
- Automatic certificate management (Let's Encrypt)
- Certificates stored in `data/acme/acme.json` (bind-mounted to `/etc/acme/acme.json` in Traefik)
- Modern TLS configuration (TLS 1.2 minimum, `sniStrict`, curated cipher suites — see `tpl/routers-http.yml.j2`)

### Rate Limiting
Can be configured via middleware in `projects/traefik.yml`:

```yaml
http:
  middlewares:
    rate-limit:
      rateLimit:
        average: 100
        burst: 50
```

Then reference in itsup-project.yml:
```yaml
ingress:
  - service: api
    middleware: [rate-limit]
```
